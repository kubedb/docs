# Rotate Authentication for HanaDB

KubeDB supports rotating HanaDB authentication credentials with a `HanaDBOpsRequest` of type `RotateAuth`.

## Before You Begin

You should be familiar with the following KubeDB concepts:

- [HanaDB](/docs/guides/hanadb/concepts/hanadb.md)
- [HanaDBOpsRequest](/docs/guides/hanadb/concepts/opsrequest.md)

## How RotateAuth Works

The RotateAuth process consists of the following steps:

1. A user creates a `HanaDB` object.
2. The KubeDB Provisioner provisions the required PetSet, services, secrets, and related resources.
3. To rotate credentials, the user creates a `HanaDBOpsRequest` with `spec.type: RotateAuth`.
4. The KubeDB Ops Manager pauses the referenced `HanaDB` object while the operation is running.
5. Ops Manager updates the HanaDB database password and the referenced authentication Secret. The password can be generated by the operator or supplied through a user-provided `kubernetes.io/basic-auth` Secret.
6. After the operation succeeds, Ops Manager resumes the `HanaDB` object.

See the [Rotate Authentication guide](/docs/guides/hanadb/rotate-auth/rotateauth.md) for a step-by-step example.
