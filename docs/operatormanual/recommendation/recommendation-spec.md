# Recommendation Spec & Status

A `Recommendation` resource represents an actionable operation generated by KubeDB Ops-manager along with its execution state.

---

## Example Recommendation

```yaml
apiVersion: supervisor.appscode.com/v1alpha1
kind: Recommendation
metadata:
  name: elastic-x-elasticsearch-x-rotate-auth-2juuee
  namespace: es
spec:
  backoffLimit: 5
  deadline: "2025-02-25T09:20:53Z"
  description: Recommending AuthSecret rotation
  operation:
    apiVersion: ops.kubedb.com/v1alpha1
    kind: ElasticsearchOpsRequest
    metadata:
      name: rotate-auth
      namespace: es
    spec:
      databaseRef:
        name: elastic
      type: RotateAuth
  recommender:
    name: kubedb-ops-manager
  rules:
    failed: has(self.status.phase) && self.status.phase == 'Failed'
    inProgress: has(self.status.phase) && self.status.phase == 'Progressing'
    success: has(self.status.phase) && self.status.phase == 'Successful'
  target:
    apiGroup: kubedb.com
    kind: Elasticsearch
    name: elastic
status:
  approvalStatus: Pending
  failedAttempt: 0
  outdated: false
  parallelism: Namespace
  phase: Pending
  reason: WaitingForApproval
```

---

## Spec Fields

* `spec.description` (string | optional) – specifies why the recommendation is generated.

* `spec.target` (TypedLocalObjectReference) – specifies the target resource (API group, kind, name) on which the recommendation will be applied.

* `spec.operation` (RawExtension) – specifies the Kubernetes resource that will be created and executed (e.g., OpsRequest).

* `spec.recommender` (ObjectReference) – specifies the component that generated the recommendation.

* `spec.deadline` (Time | optional) – specifies the execution deadline for the recommendation.

* `spec.requireExplicitApproval` (bool | optional) – specifies whether manual approval is required. If `true`, ApprovalPolicy is ignored.

* `spec.rules` (OperationPhaseRules) – specifies how operation execution state is determined using CEL expressions.

---

## OperationPhaseRules

Frist have a look on `operation` yaml,

```shell
$ kubectl get recommendation -n es elastic-x-elasticsearch-x-update-version-2juuee -o jsonpath='{.spec.operation}' | yq -y
apiVersion: ops.kubedb.com/v1alpha1
kind: ElasticsearchOpsRequest
metadata:
  name: update-version
  namespace: es
spec:
  databaseRef:
    name: elastic
  type: UpdateVersion
  updateVersion:
    targetVersion: xpack-9.2.3
status: {}
```

* `spec.rules.success` (string) – specifies the rule to detect successful execution.

* `spec.rules.inProgress` (string) – specifies the rule to detect ongoing execution.

* `spec.rules.failed` (string) – specifies the rule to detect failed execution.

**Note:**

* `self` refers to the **Operation resource created from `spec.operation`**
* expressions are evaluated against the **operation's status**

---

## Status Fields

* `status.approvalStatus` (ApprovalStatus) – specifies whether the recommendation is `Pending`, `Approved`, or `Rejected`.

* `status.phase` (RecommendationPhase) – specifies the current lifecycle phase (`Pending`, `Waiting`, `InProgress`, `Succeeded`, `Failed`, `Skipped`).

* `status.reason` (string) – specifies a message describing the current state.

* `status.reviewer` (Subject | optional) – specifies who approved or rejected the recommendation.

* `status.comments` (string | optional) – specifies reviewer comments.

* `status.reviewTimestamp` (Time | optional) – specifies when the review occurred.

* `status.approvedWindow` (ApprovedWindow | optional) – specifies when the recommendation is allowed to execute.

* `status.parallelism` (Parallelism) – specifies execution concurrency control (`Namespace`, `Target`, `TargetAndNamespace`).

* `status.outdated` (bool) – specifies whether the recommendation is outdated and should not be executed.

* `status.conditions` ([]Condition | optional) – specifies the current conditions of the recommendation.

* `status.createdOperationRef` (LocalObjectReference | optional) – specifies the created operation resource.

* `status.failedAttempt` (int32) – specifies how many times execution has failed.

* `status.observedGeneration` (int64 | optional) – specifies the last observed generation of the resource.

---

## ApprovedWindow Fields

* `status.approvedWindow.window` (WindowType) – specifies execution strategy (`Immediate`, `NextAvailable`, `SpecificDates`).

* `status.approvedWindow.maintenanceWindow` (TypedObjectReference | optional) – specifies the referenced MaintenanceWindow.

* `status.approvedWindow.dates` ([]DateWindow | optional) – specifies allowed execution time ranges.
